# Platform division

DC Hub splits work between **web** (identity, portal, users) and **desktop** (pipeline, filesystem, cloud OAuth). Supabase is the source of truth for shared data; each field group has one canonical admin surface.

## Field ownership

| Layer | Fields | Admin surface |
|-------|--------|---------------|
| **Portal** | `slug`, `portal_bg`, `domain_whitelist`, `website`, `initials` | Web `/` client drawer (admin) |
| **Branding** | `logo_url`, `accent` | Web — logo on CDN at `branding/{client_id}/` |
| **Storage** | `R2_BUCKET`, `R2_PUBLIC_DOMAIN` | Environment edge secrets — not per client |
| **Taxonomy labels** | `dimension_labels` (entity/angle/format display names) | Web client drawer (admin) |
| **Tags** | `tags` tree + `shortcode` | Web tag admin; DB is source of truth |
| **Local-only** | folder paths, OAuth tokens | Desktop Settings / Pipeline |

## Rules

- Never edit the same DB column from web and desktop.
- Desktop reads portal/branding fields; edits via web portal link.
- Object keys in R2 are prefixed `{client_id}/` within the environment bucket.
- Rename tasks queue filesystem updates when tags change on web (desktop picks up on next run).
