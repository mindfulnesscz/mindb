# Admin Dashboard

Accessible at `/` for users with `editor` or `admin` role. Client identity, users, branding, and tags are owned by the web — see [Platform division](/getting-started/platform-division).

## Access rules

| Role | What they see |
|---|---|
| `admin` | Clients tab + Users tab; can create/edit clients, logos, tags, dimension labels |
| `editor` | Clients tab (read-oriented); redirected to sole client portal when they have exactly one |
| `member` / `public` | "Staff access only" screen |

Desktop pipeline access is limited to **editor** and **admin**.

## Clients tab

Shows clients as cards (badge, name, slug, website, domain whitelist preview).

- **Click a card** → portal (`/:slug`)
- **Edit** (admins) → client drawer

### Client drawer fields

| Field | Notes |
|---|---|
| Name | Required |
| Initials | Max 3 chars |
| Brand colour | Hex |
| Logo | File upload to CDN (`branding/{client_id}/`) via `r2-branding-upload`, or paste URL |
| Taxonomy labels | Per-client display names for entity / angle / format (`dimension_labels`) |
| Website | Optional |
| Portal URL slug | Shareable portal path |
| Portal background | Colour, gradient, or image URL |
| Domain whitelist | Auto-assigns matching emails as `member` of this client |
| Tags | CRUD tree + shortcodes (DB source of truth) when editing an existing client |

## Users tab (admin only)

Table of all profiles:

- Name, email
- **Access** — for `member`: client dropdown; for `editor`: client checkboxes (`client_members`)
- **Role** — public / member / editor / admin (cannot change your own role)

Saves via `update_user_access` (not only `update_user_role`).

## Settings

`/settings` stores optional portable Supabase URL + anon key in `localStorage` when env vars are absent.
