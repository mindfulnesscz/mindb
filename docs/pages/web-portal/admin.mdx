# Portal administration

The root route is the staff landing. Editors/admins navigate client cards. Client forms (admin) manage name, initials, accent, logo (CDN upload or URL), `dimension_labels`, site, slug, portal background, domain whitelist, and the **tags** tree. Users load via `get_all_profiles`; administrators change role and access through `update_user_access` (member client picker, editor `client_members` checkboxes). Desktop sign-in remains editor/admin only.

See [Admin dashboard](/admin) and [Platform division](/getting-started/platform-division).
