# Assets, children, and variants

An asset row has client, name/shortcode, taxonomy arrays, status, permission, version/latest flags, thumbnail/download URLs, timestamps, and relationship fields.

Two grouping systems coexist:

- Legacy galleries use `parent_id`; child shortcodes historically use `Parent|childStem`.
- Stable package variants use `variant_of`; one primary row is shown as the top-level card and siblings appear in detail.

Top-level queries exclude both relationship types and separately count/load them. Archived/disconnected related rows are excluded. Deleting a referenced primary may be blocked when foreign-key behavior does not cascade or null the relationship.

Status and permission are separate. Status describes lifecycle (`draft`, `review`, `approved`, `published`, `archived`, `disconnected`); `perm` describes audience (`public`, `client`, `internal`).
