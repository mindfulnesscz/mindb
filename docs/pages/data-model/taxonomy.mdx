# Taxonomy

Internal tag dimensions are always **entity**, **angle**, and **format** (machine keys). Per-client **display names** live in `clients.dimension_labels`.

**Source of truth** is `public.tags`. For the full edit/sync workflow (portal vs desktop), start with [Tags and export destinations](/getting-started/tags-and-destinations).

## Where to edit

| What | Who | Surface |
|---|---|---|
| Dimension labels | Admin | Portal client drawer |
| Parent groups (no `shortcode`) | Admin / editor | Portal **Tags** only |
| Leaf tags (have `shortcode`) | Admin / editor | Portal **Tags**, or desktop Vocabulary + Sync |
| Filename tokens in use | Operators | Must match leaf shortcodes — renaming a shortcode does not rename files by itself |

Desktop Vocabulary **Sync** publishes dirty **leaves** then reloads from the DB. It never creates parent groups. Parent choice on desktop is a select of groups already in the portal.

## Import / export JSON

Templates are **local JSON files** (not rows in the database). On create client, optionally attach a file; on an existing client use **Import from JSON** or **Export JSON** in Tags.

Sample file in the portal app: `web/apps/client-hub/public/taxonomy.sample.json`.

```json
{
  "version": 1,
  "dimension_labels": { "entity": "Entity", "angle": "Angle", "format": "Format" },
  "nodes": [
    { "key": "entity.group", "dimension": "entity", "name": "Group", "parent_key": null },
    { "key": "entity.group.leaf", "dimension": "entity", "name": "Tag", "parent_key": "entity.group", "shortcode": "Sam" }
  ]
}
```

- **`key`** — stable identity; also used as the Obsidian tag on desktop export
- **`parent_key`** — tree grouping: nodes **without** `shortcode` are **parent groups** (portal-managed); leaves have a `shortcode`
- **`shortcode`** — filename token (leaves only)

Parse/validate/import: `taxonomyImport.ts` (`parseAndValidateTaxonomyJson`, `importTaxonomyToClient`). Export rebuilds the same document from the live DB (`buildTaxonomyDocument` / **Export JSON**). Desktop loads shortcoded tags from the DB for filename parsing and groups them by parent.

Renaming a leaf’s **display name** on the portal also rewrites that label on the client’s asset rows (`name`, `entities` / `angles` / `formats`, `tags`) so the gallery updates without waiting for a desktop run. Shortcode renames still enqueue `rename_tasks` for filesystem updates.

See [Platform division](/getting-started/platform-division) and [Naming and vocabulary](/desktop/naming).
