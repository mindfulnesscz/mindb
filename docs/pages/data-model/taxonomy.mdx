# Taxonomy

Internal tag dimensions are always **entity**, **angle**, and **format** (keys). Per-client **display names** live in `clients.dimension_labels`.

**Source of truth** is `public.tags`. Admins edit them in TagsAdmin or import a local JSON file.

## Import from JSON

Templates are **local JSON files** (not rows in the database). On create client, optionally attach a file; on an existing client use **Import from JSON** in Tags.

Sample file: [`/taxonomy.sample.json`](/taxonomy.sample.json) in the portal app (`web/apps/client-hub/public/taxonomy.sample.json`).

```json
{
  "version": 1,
  "dimension_labels": { "entity": "Entity", "angle": "Angle", "format": "Format" },
  "nodes": [
    { "key": "entity.group", "dimension": "entity", "name": "Group", "parent_key": null },
    { "key": "entity.group.leaf", "dimension": "entity", "name": "Tag", "parent_key": "entity.group", "shortcode": "Sam" }
  ]
}
```

- **`key`** — stable identity; also used as the Obsidian tag on desktop export
- **`parent_key`** — tree grouping (replaces legacy desktop subtypes); groups have no shortcode, leaves do
- **`shortcode`** — filename token (leaves only)

Parse/validate/import: `taxonomyImport.ts` (`parseAndValidateTaxonomyJson`, `importTaxonomyToClient`). Desktop loads shortcoded tags from the DB for filename parsing and groups them by parent.
