# Settings reference

| Setting | Effect |
|---|---|
| Source folder | Root scanned for `OUT` trees |
| Target folder | Local mirrored publishing destination |
| Vault folder | Generated Obsidian notes |
| Generate thumbnails | Local sidecars and, with R2, thumbnail upload |
| Upload originals to CDN | Independent R2 original stage |
| Distribute | Pull sibling deliverables into package folders |
| Publish | Mirror to local target and flag removals |
| Flat/cloud export | Provider API export toggle |
| Obsidian | Note generation |
| Dry run | Intended for filesystem actions; verify stage behavior before assuming no network writes |
| Highest version only | Drops older version candidates during distribution |
| Package prefix | Package discovery prefix; default `[00] 📦` |
| OUT folder | Publish-root name; default `[03] OUT` |
| Filter mode/marks | Blacklist exclusion or whitelist inclusion |
| Thumbnail width/quality | WebP resize width and encoder quality |
| Identity migrated | Enables manifest/stable-ID matching; set only after migration |

Destination fields are detailed in [Provider comparison](/cloud-storage/overview). OAuth tokens are machine-local secrets.

## Environments and R2

Desktop **Settings → Environment** holds URL + anon key per environment (Production / Staging / Local). There is no permanent R2 credential UI on the desktop.

CDN uses environment edge secrets (`R2_BUCKET`, `R2_PUBLIC_DOMAIN`, Cloudflare parent credentials) and per-run `r2-grant`. See [R2 CDN publishing](/desktop/cdn) and [Team staging setup](/getting-started/team-dev-setup).
