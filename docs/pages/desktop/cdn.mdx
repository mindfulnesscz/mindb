# Cloudflare R2 setup and publishing

DC Hub uses Cloudflare R2 as the public CDN for thumbnails and, when enabled, original files. Storage is **environment-level**: one bucket and public domain per Supabase project (Production / Staging / Local), not per client. Object keys are prefixed `{client_id}/` so many clients share one bucket safely.

> R2 links are public bearer links. Anyone who knows a URL can download its object. Do not use this setup for confidential assets without adding a private delivery layer. See [Cloud security](/cloud-storage/security).

The desktop holds **no permanent R2 credentials**. Each pipeline run requests a short-lived grant from the `r2-grant` edge function. Logos are uploaded from the web admin via `r2-branding-upload` to `branding/{client_id}/logo.ext`.

## Before you start

You need:

- a Cloudflare account with R2 enabled;
- one R2 bucket per environment (e.g. `dc-hub-staging`, production bucket);
- for production: a custom CDN hostname; for staging/dev: the `r2.dev` public URL is fine;
- edge function secrets on the matching Supabase project (see below).

Day-to-day development should use **shared Staging** for both Supabase and R2 — see [Team staging setup](/getting-started/team-dev-setup). Do not mix a local database with a shared bucket.

## 1. Create the environment bucket

1. Open **Cloudflare Dashboard → R2 object storage**.
2. Create a bucket named for the environment (e.g. `dc-hub-staging`).
3. Keep the exact name for the `R2_BUCKET` secret.

## 2. Public domain

**Staging / rehearsal:** enable the bucket’s **r2.dev** public subdomain. Use that URL as `R2_PUBLIC_DOMAIN` (no trailing slash), e.g. `https://pub-xxxx.r2.dev`.

**Production:** connect a custom domain under the bucket’s **Custom Domains**, wait until Active, and set `R2_PUBLIC_DOMAIN` to that origin (`https://cdn.example.com`). Leave trailing slashes off.

Browser-facing URLs look like:

```text
https://cdn.example.com/{client_id}/thumbnails/...
https://cdn.example.com/branding/{client_id}/logo.png
```

## 3. Configure browser CORS

Portal downloads fetch the public CDN URL. Allow cross-origin `GET` and `HEAD`:

```json
[
  {
    "AllowedOrigins": ["*"],
    "AllowedMethods": ["GET", "HEAD"]
  }
]
```

Until CORS is configured, downloads may fall back to opening the URL in a new tab.

## 4. Provision the Control API (edge secrets)

Create an R2 **Account API token** with object read/write (or Admin Read & Write) scoped as needed. From the creation screen and R2 overview, set these on the Supabase project (Dashboard → Edge Functions → Secrets, or CLI):

| Secret | Source |
|--------|--------|
| `R2_BUCKET` | Bucket name |
| `R2_PUBLIC_DOMAIN` | Public origin (no trailing slash) |
| `CF_API_TOKEN` | Token value (bearer for Cloudflare REST) |
| `CF_ACCOUNT_ID` | R2 account ID |
| `R2_PARENT_ACCESS_KEY_ID` | Access Key ID from the same token create screen |

```bash
supabase link --project-ref <env-ref>
supabase secrets set R2_BUCKET=… R2_PUBLIC_DOMAIN=… CF_API_TOKEN=… CF_ACCOUNT_ID=… R2_PARENT_ACCESS_KEY_ID=…
supabase functions deploy
```

Local stack: put the same values in `supabase/functions/.env` (gitignored). With `supabase start`, the edge runtime container loads that file. If CDN returns **503 `name resolution failed`**, the edge runtime container is down (common after Docker Desktop / Mac sleep) — restart it (`docker start supabase_edge_runtime_dc-hub` or `npx supabase stop && npx supabase start`). A 503 body about missing `R2_BUCKET` / Cloudflare secrets means the `.env` is incomplete. Until provisioned, pipeline logs `CDN steps disabled — Storage grant refused (503)` and skips uploads.

Do **not** paste Access Key / Secret into desktop Settings. The desktop never stores parent R2 keys.

## 5. Verify publishing

1. Sign the desktop into the matching environment as editor/admin.
2. Enable **Generate thumbnails** or **Upload originals to CDN**.
3. Run the pipeline against a small test asset.
4. Confirm R2 contains keys under `{client_id}/thumbnails/` or `{client_id}/originals/`.
5. Open the public URL; confirm portal download works. Gallery URLs should include `?v=<hash>`.

## How publishing works

When thumbnails are enabled and the grant succeeds, the pipeline uploads via native Rust (`upload_to_r2`): SHA-256 content check, skip if unchanged, set `Cache-Control: public, max-age=31536000, immutable`. Public URLs stored in Supabase include `?v=<content-hash>` so version bumps that overwrite a stable object key still change the gallery URL and bust browser cache. Originals have a separate toggle. Local mtime/size cache and content-hash dedupe skip unnecessary PUTs; a DB inventory URL alone does **not** skip re-verification.

## Object keys

Keys always start with the client UUID (or `branding/{client_id}/` for logos):

| Identity mode | Thumbnail | Original |
|---|---|---|
| Migrated package | `{client_id}/thumbnails/<stableId>/<childId>.webp` | `{client_id}/originals/<stableId>/<childId><ext>` |
| Legacy/fallback | `{client_id}/thumbnails/<full-stem>-thumb.webp` | `{client_id}/originals/<version-stripped-shortcode><ext>` |

Public URLs append `?v=<12-char content hash>`. After an original upload, sibling objects with the same key prefix but a stale extension may be removed. Whole-bucket orphan cleanup is not an automatic UI action.

## Troubleshooting

| Symptom | Check |
|---|---|
| **CDN steps disabled / grant 503** | Edge secrets set and functions deployed; `R2_BUCKET` / `R2_PUBLIC_DOMAIN` present. Locally: edge runtime container running + `supabase/functions/.env` |
| **503 `name resolution failed` (local)** | `supabase_edge_runtime_*` exited — restart edge runtime / `supabase start` |
| **Public URL is 404/403** | Domain Active, object key has `{client_id}/` prefix, domain has no trailing slash |
| **Download opens in a new tab** | CORS `GET`/`HEAD` from `*`; purge CDN cache after CORS changes |
| **Gallery shows old thumbnail after version bump** | Re-run CDN + sync so `thumbnail_url` gets a new `?v=` hash; hard-refresh only needed for URLs written before cache-busting |
| **Logo upload 503** | Same secrets; `r2-branding-upload` deployed; user is **admin** |
| **Unexpected `//thumbnails` URL** | Remove trailing slash from `R2_PUBLIC_DOMAIN` |

See also [Platform division](/getting-started/platform-division) and [Database environments](/operations/database-environments).
