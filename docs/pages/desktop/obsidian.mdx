# Obsidian DAM output

The Obsidian stage generates markdown notes in the configured vault. Notes are derived artifacts built from scanned assets, parsed vocabulary, inherited taxonomy, versions, CDN/provider URLs, and configured depth. This phase does not upload to R2 or create Supabase rows.

Its strength is a human-readable, regenerable knowledge base that can be browsed and linked independently. Its weakness is duplication: notes can be stale if a run stops before regeneration, and manual edits may be overwritten depending on generator behavior. Treat source files and Supabase as operational authority; treat generated notes as a projection unless your team establishes a manual-edit policy.
