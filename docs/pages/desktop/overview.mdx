# Desktop application

The desktop app is a Tauri 2 application: React/TypeScript owns orchestration and UI state; Rust performs operations that need native processes, unrestricted binary I/O, a local OAuth listener, or AWS Signature V4 signing.

Client-specific **machine-local** configuration includes source/target/vault folders, cloud destination tokens, and the stable-identity migration flag (DB). Portal identity, branding, and tags are web-owned ([Platform division](/getting-started/platform-division)). Environments hold Supabase URL + anon key only; CDN uses `r2-grant`.

Global pipeline settings define stage toggles, folder markers, filters, thumbnail parameters, and run options.

Native commands include thumbnail generation, the OAuth/auth callback listener, R2 upload/list/delete (with grant credentials), Dropbox/OneDrive operations, and a native Supabase request bridge. See [Code map](/reference/code-map).

The app does not provide a distributed job queue or transaction spanning providers. A run is an ordered collection of best-effort operations; errors are logged and most per-file loops continue. This makes partial success normal and makes the [publishing runbook](/operations/runbook) important.
