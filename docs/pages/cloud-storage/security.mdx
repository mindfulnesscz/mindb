# Cloud integration security

> **Current residual risk:** provider OAuth tokens (and Google client secret where used) remain in machine-local client configuration. Supabase user authentication, secret-free exports, and R2 grants are **already shipped** — permanent service-role and parent R2 keys do not belong on the desktop. See [Auth & secrets roadmap](/desktop/authentication-plan).

## Public CDN nature

R2 objects served through a public domain or custom hostname are reachable by anyone who knows the URL. Treat CDN assets as published unless a private delivery layer is added later.

## Local secrets today

Expect only **OAuth tokens** (and optional Google client secret) in local config. Restrict OS account access, disk backups, logs, and screenshots. Never put a service-role key or parent R2 secret in web env vars or the desktop. Client export bundles are secret-free (`_version: 2.0`); old imports strip legacy credential fields and warn to rotate.

## Temporary grants

Pipeline CDN uploads use short-lived credentials from `r2-grant` (role + client scope). Branding uploads use `r2-branding-upload` (admin only). Failures when unprovisioned are explicit 503s — they do not fall back to embedding permanent keys.

## Operator habits

Prefer Staging for multi-person work ([Team staging setup](/getting-started/team-dev-setup)). Rotate provider tokens after suspected exposure. Prefer least-privilege Cloudflare tokens for edge secrets.
