# Settings reference

| Setting | Effect |
|---|---|
| Source folder | Root scanned for `OUT` trees |
| Target folder | Local mirrored publishing destination |
| Vault folder | Generated Obsidian notes |
| Generate thumbnails | Local sidecars and, with R2, thumbnail upload |
| Upload originals to CDN | Independent R2 original stage |
| Distribute | Pull sibling deliverables into package folders |
| Publish | Mirror to local target and flag removals |
| Flat/cloud export | Provider API export toggle |
| Obsidian | Note generation |
| Dry run | Intended for filesystem actions; verify stage behavior before assuming no network writes |
| Highest version only | Drops older version candidates during distribution |
| Package prefix | Package discovery prefix; default `[00] 📦` |
| OUT folder | Publish-root name; default `[03] OUT` |
| Filter mode/marks | Blacklist exclusion or whitelist inclusion |
| Thumbnail width/quality | WebP resize width and encoder quality |
| Identity migrated | Enables manifest/stable-ID matching; set only after migration |

Destination fields are detailed in [Provider comparison](/cloud-storage/overview). Credential fields are secrets even when displayed as ordinary configuration.

## Cloudflare R2 fields

R2 configuration belongs to the active client. Use the S3 API endpoint and S3 Access Key ID/Secret Access Key—not a general Cloudflare API token. The public domain is the active browser-facing custom domain and is different from the authenticated S3 endpoint.

| Setting | Format |
|---|---|
| R2 Endpoint | `https://<ACCOUNT_ID>.r2.cloudflarestorage.com` or its jurisdiction-specific equivalent; no bucket path |
| Bucket name | Exact client-specific R2 bucket name |
| Public domain | `https://cdn.example.com`; no path or trailing slash |
| Access Key ID | ID generated by an R2 Object Read & Write token |
| Secret Access Key | Secret shown once when the R2 token is created |

See [Cloudflare R2 setup and publishing](/desktop/cdn) for bucket creation, custom-domain, CORS, credential, testing, and troubleshooting steps.
